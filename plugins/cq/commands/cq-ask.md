# /cq:ask

向 cq 知识库提问，获取基于已有知识单元的回答。

## 参数

用户在 `/cq:ask` 后提供的文本即为问题。例如：

```
/cq:ask webpack 5 中 Node.js polyfills 怎么处理？
```

## 指令

### 第 1 步 — 解析问题

从用户的问题中提取：

- **领域标签** — 识别问题涉及的技术、工具、框架或层次，构造 2–5 个领域标签。
- **上下文** — 如果问题中提及了特定语言、框架或模式，提取为 `context` 字段（`language`、`framework`、`pattern`）。
- **关键词** — 问题中的核心术语，用于辅助判断结果相关性。

### 第 2 步 — 查询知识库

使用提取的信息调用 `query` MCP 工具：

```
query(
  domain=<领域标签列表>,
  context={ language: <语言>, framework: <框架>, pattern: <模式> },
  limit=10
)
```

如果问题范围较广，可以进行多次查询，使用不同的领域标签组合以覆盖更多相关知识。

如果工具调用失败（MCP 服务器不可用、超时或任何错误），告知用户知识库暂时不可用，并建议稍后重试。

### 第 3 步 — 综合回答

#### 有相关结果时

根据返回的知识单元，综合给出回答。格式如下：

```
## 回答

{基于知识单元综合的回答。直接回答用户的问题，将多个相关知识单元的信息整合为连贯的答案。}

### 相关知识单元

- **{summary}**（可信度: {confidence}）
  {action}

- ...
```

回答时注意：

- 以 `action` 字段为核心，给出具体可操作的建议。
- 结合 `detail` 字段提供背景和原因。
- 如果多个知识单元相关，按相关性排序，整合为连贯的回答。
- 对于可信度较低（< 0.5）的知识单元，明确提示用户需要额外验证。
- 如果知识单元之间存在矛盾，指出差异并建议用户自行验证。

#### 无相关结果时

```
知识库中未找到与该问题直接相关的知识。

**建议：**
- 尝试用不同的关键词重新提问。
- 如果你在后续工作中发现了相关知识，可以使用 `/cq:reflect` 将其提交到知识库。
```

### 第 4 步 — 确认有用的知识

如果用户表示某个知识单元对其有帮助，使用该单元的 `id` 调用 `confirm` 以增强其可信度。

## 边界情况

- **问题过于宽泛** — 如果问题太笼统无法构造有效的领域标签，请用户提供更具体的技术细节。
- **多次查询** — 对于跨领域的复杂问题，可以分多次查询并合并结果。
- **结果过多** — 仅展示最相关的前 5 个知识单元，其余在末尾注明"还有 N 个相关知识单元，如需查看请告知"。
